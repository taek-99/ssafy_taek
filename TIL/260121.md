# RAG 구조와 생성형 AI

## LLM 한계 극복
- 프롬프트 엔지니어링: 특정 작업에 대해 모델이 더 잘 반응하도록 입력 텍스트를 최적화하는 방식
- RAG: 입력 프롬프트와 검색 기반의 정보를 결합하여 증강된 정보를 기반으로 답변을 생성하도록 하는 방식
- 파인튜닝: 사전 훈련된 모델을 특정 작업이나 데이터셋에 맞게 추가적으로 조정하는 방식

## RAG
- Retrieval(검색): 외부 데이터 및 소스를 검색하여 정보 획득
- Augmented(증강): 사용자의 질문을 보강하여 보다 정확한 문맥 제공
- Generation(생성): 향상된 정보를 기반으로 더 좋은 답변 생성
- 답변할 때 확실한 출처를 기반으로 생성하게 됨

### RAG의 장점
- 환각 현상 감소
- 도메인 적응성 개선
- Open domain QA 성능 향상
- 참고한 Knowledge base가 적절한지 판단 가능
- 정보 검색에 강함

### RAG 이론
- 정보검색
    - 필요한 데이터를 검색하는 작업
    - 데이터베이스, 인터넷, 또 다른 정보 저장소에서 관련 정보를 찾아내는 과정

- 역색인
    - 색인:1 -> 1페이지 호출, 100 -> 100페이지 호출
    - 각 데이터에 빠르게 접근할 수 있도록 도움

- TF-IDF
- BM25

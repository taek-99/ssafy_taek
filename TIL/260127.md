# 데이터 엔지니어링 도커 개념 및 활용

## 챕터
- 컨테이너와 도커
- Docker Desktop과 설치
- Docker 기본 명령어
- Docker compose 실습

## 컨테이너와 도커

### 도커
- 애플리케이션을 컨테이너라고 하는 가볍고 독립적인 환경에서 실행할 수 있도록 도와주는 오픈소스 가상화 플랫폼

### 컨테이너
- 컨테이너는 어떤 물체나 데이터를 외부와 분리된 공간에 담아두는 것

### 컨테이너의 필요성
- 환경 차이 없이 어디서든 실행 가능
- 설치/ 설정 충돌 방지
- 빠른 실행과 이식성

### 도커 핵슴 구성 요소
- 도커 엔진
  - 도커의 중심이 되는 실행 시스템
  - 컨테이너를 실제로 생성하고 실행하는 역할을 맡음
  - 도커 데몬에서 도커 API 요청을 받아 처리

- 도커 클라이언트
  - 우리가 터미널에서 입력하는 도커 명령어가 여기에 해당
  - 사용자가 입력하는 명령은 도커 클라이언르로 들어강

- 도커 객체
  - 도커에서 다루는 주요 구성 단위
  - Image
  - Container
  - Service

- 도커 레지스트리
  - 이미지를 저장하고 공유할 수 있는 중앙 저장소
  - 가장 대표적인 예로는 Docker Hub

- 도커 파일
  - 도커 파일은 이미지를 만들기 위한 설계도
  - 우리가 원하는 환경과 명령들을 스크립트 형식으로 작성한다

- 도커 이미지
  - 이미지는 하나의 정적인 파일로 실행할 준비가 된 상태의 환경
  - docker build 명령을 통해 Dockerfile로부터 이미지를 생성

- 도커 컨테이너
  - 컨테이너는 이미지의 복사본이 메모리에서 실행중인 상태
  - 이미지를 docker run하면 컨테이너가 생성

- 도커 컴포즈
  - 여러 개의 컨테이너를 한번에 실행하고 관리할 수 있는 도구
  - docker-compose.yml 파일에 컨테이너 구성과 관계를 정의

docker compose up 실행
docker compose ps 서비스 확인
docker compose down 끄기

## 컨테이너 기술과 도커 개요

### 도커
- 도커
  - 애플리케이션을 컨테이너라는 가볍고 독립적인 환경에서 실행할 수 있도록 도와주는 오픈소스 가상화 플랫폼

- 컨테이너
  - 어떤 물체나 데이터를 외부와 분리된 공간에 담아두는것
  - 필요성
    - 환경 차이 없이 어디서든 실행 가능
    - 설치/ 설정 충돌 방지
    - 빠른 실행과 이식성

### 도커의 핵심 구성 요소
- 도커 엔진
  - 도커의 중심이 되는 실행 시스템
  - 컨테이너를 실제로 생성하고 실행하는 역할을 맡음

- 도커 클라이언트
  - 우리가 터미널에 입력하는 docker 명령어
  - 입력한 명령어를 도커엔진에 전달

- 도커 객체
  - 도커에서 이루는 주요 구성 단위
  - 이미지: 컨테이너를 만들기 위한 설계도
  - 컨테이너: 이미지를 실제로 실행한 실체
  - 서비스: 여러 개의 컨테이너를 묶어 하나의 서비스처럼 운영하는 단위

- 도커 레지스트리
  - 이미지를 저장하고 공유할수 있는 중앙 저장소

- 도커 파일
  - 이미지를 만들기 위한 설계도

### 도커 파일, 도커 이미지, 도커 컨테이너 관계
- 도커 이미지
  - 이미지는 하나의 정적인 파일로, 실행할 준비가 된 상태의 환경
  - 도커 빌드를 명령을 통해 도커파일로부터 이미지를 생성

- 도커 컨테이너
  - 컨테이너는 이미지의 복사본이 메모리에서 실행중인 상태


### 도커 관련 주요 도구
- 도커 컴포즈
  - 여러 개의 컨테이너를 한번에 실행하고 관리할 수 있는 도구

## 데이터 파이프라인

### 데이터
- 정보
  - 실제 도움이 되는 데이터
  - 유의미
  
- 데이터
  - 2010년대 이후를 대표하는 키워드
  - 대규모로 수집된 원시 자료
  
- 데이터가 중요해진 이유
  - 빅데이터: 데이터의 양, 다양성, 증가속도, 신뢰성, 가치
  
### 데이터 엔지니이어의 주요활동
- 주요역할
  - 데이터를 안정적으로 수집하고 가공하여 전달
  - 분석과 모델링이 가능하도록 데이터 흐름을 자동화
  - 신뢰성 있고 재사용 가능한 파이프라인 구축
  
- 주로 하는 일
  - 다양한 시스템에서 데이터 수집
  - 정제 및 변환

### 데이터 파이프라인
- 데이터를 추출하고 정제하고 저장, 분석, 시각화하는 일련의 자동화 과정
  
### OLAP와 OLTP
- OLTP
  - 운영 데이터 처리 시스템
  - 실시간 트랜잭션 처리
  - 행 단위 저장 구조
  - 빠른 입력, 수정, 삭제에 최적화
  
- OLAP
  - 분석 데이터 처리 시스템
  - OLTP 등에서 수집된 데이터를 기반으로 통계, 리포트 분석
  - 컬럼 기반 저장 구조를 사용

### ETL의 구조
- 데이터를 가공한 후 저장 전통적인 방식
- 추출-> 가공 -> 저장

#### 데이터 처리 방식 - 배치와 스트리밍
- 베치 처리 방식
  - 데이터를 모아서 한번에 처리하는 방식
  - 주로 하루 1회, 또는 시간 단위로 처리가 이루어짐
  - 정확성과 대량 처리에 적합함

- 데이터 스트리밍 처리 방식
  - 데이터가 들어오는 즉시 실시간 처리하는 방식
  - 빠르게 변화하는 데이터에 즉시 반응 가능
  
- 배치는 정확성과 안정성 중심, 스트리밍은 실시간성과 즉시성 중심
- 데이터의 속도, 목표에 따라 적절한 방식을 선택

#### 데이터 파이프라이의 기본 구조
- 자동화된 데이터 흐름
  - 데이터 소스, 수집, 가공, 저장, 분석/제공

### 데이터 저장소의 중요성
- 저장소는 분석을 위한 인프라
  - - 데이터를 단순히 저장하는 것이 아니라 분석, 활요응ㄹ 위한 반응 설계 필

### 데이터 레이크의 구성
- DW의 구조적 질서에 유연성을 결합
- 분석, 모델링, BI 모두 대응하는 형태

### 데이터 수집 도구
- Kafka
  - 분석 메시지 큐 시스템
  - 대용량 데이터를 빠르고 안정적으로 전달
  
### 데이터 처리 도구
- Spark
  - 대규모 배치 처리 프레임워크
  - ETL/머신러닝 통합 가능

- Flink
  - 스트리밍 처리 전문 프레임워크
  - 이벤트 기반 실시간 분석에 최적화
  
### 데이터 저장 도구
- RDBMS
- Elasticsearch
- Hadoop

### 데이터 모니터링 및 워크플로우 관리 도구
